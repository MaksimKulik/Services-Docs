**Balance**
The amount of funds available for a service in a specific currency. Balances are used to support financial operations and may be updated manually or synchronized with a provider.

**Credential**
A provider-specific value required to authenticate and authorize a service. Examples include API keys, secret keys, merchant IDs, or access tokens.

**Currency**
A monetary unit used for transactions and balances. Each balance and fee profile is configured for a specific currency.

**Fee component**
An individual rule within a fee profile that defines how a fee is calculated for a specific transaction type, such as deposits or withdrawals.

**Fee profile**
A set of fee rules applied to transactions processed by a service. Fee profiles are usually configured per currency and may include multiple fee components.

**Provider**
An external system or payment processor that handles transactions on behalf of the platform. Providers define credential requirements and supported capabilities.

**Rolling reserve**
A portion of funds that is temporarily withheld for a defined period. Rolling reserves are commonly used to mitigate financial risk.

**Service**
A configured integration that connects the platform to a specific provider. Services define how transactions are processed, authenticated, and managed.

**Service status**
The current operational state of a service, such as active or inactive. Only active services are typically used for processing transactions.

**Settlement**
The process of transferring funds after transactions are completed. Settlement rules define timing, delays, and applicable fees.

**Validation message**
A system message displayed when configuration rules are not met. Validation messages usually indicate missing, incorrect, or incompatible values.
