    This documentation describes the Admin UI behavior. Any API examples are illustrative and contain no sensitive data.

    What this documentation covers¶

    A Service is a configuration unit that connects a payment Provider with the required Credentials and operational settings. In this section you will learn how to create, validate, and maintain services in a safe and predictable way.

    Security and compliance
    Do not place real card data (PAN, CVV), full bank account numbers, or production secrets in screenshots, examples, or logs. Use masked values and placeholders.
